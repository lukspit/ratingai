# Rating.ai: Visão Estratégica, Design e Precificação

## 1. O Conceito: "Serene Tech" para o Setor Jurídico
O setor tributário e jurídico exige **confiança e autoridade**. O design não deve ser "gamificado" ou excessivamente "hi-tech" (neon, ciberpunk). Deve ser o que chamamos de **Serene Tech**:
- **Paleta:** Navy Deep (Confiança), Emerald (Sucesso/Recuperação), Ouro (Premium/Valor).
- **Tipografia:** Serifada para títulos (autoridade) e Sans-serif moderna para dados (clareza).
- **Sensação:** "Este sistema é mais preciso que um humano e tão sério quanto um juiz."

---

## 2. A Jornada do Herói Tributário (UX Flow)

### Fase 1: O "Input" Sólido
- **Ação:** Drag and drop de PDFs. 
- **UX:** O sistema identifica o tipo de documento em tempo real (ex: "DRE Identificada", "Balanço Patrimonial Detectado").
- **Valor:** Isso remove a ansiedade do usuário de "será que o sistema vai ler certo?".

### Fase 2: A "Caixa Preta" Processando
- **Visual:** Micro-animações de "extração de evidências". 
- **Feedback:** "Analisando 1.240 lançamentos contábeis...", "Identificando despesas não recorrentes...", "Calculando PLRA conforme Portaria 6.757".
- **Estratégia:** O usuário precisa sentir que o sistema está fazendo um trabalho pesado que ele levaria horas para fazer. O tempo de espera (mesmo que artificial) aumenta o valor percebido.

### Fase 3: O "Money Shot" (Resultado)
- **Visual:** Um comparativo visual impactante.
    - **Antes:** Rating A (0% de desconto) - Dívida: R$ 1.000.000.
    - **Depois:** Rating D (70% de desconto) - Dívida: R$ 300.000.
- **Entregável:** Um botão de "Gerar Laudo PDF" com a fundamentação jurídica pronta para o portal REGULARIZE.

## 4. O Diferencial Estratégico
O Rating.ai não é apenas uma ferramenta de análise de arquivos. É um **Gerador de Economia Real**. 
A comunicação deve focar menos em "IA" e mais em "Maximização de Desconto Legal".
